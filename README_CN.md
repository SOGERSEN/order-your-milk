## 🎚️ 八档执行系统

根据任务重要性选择投入：

| 档位 | 名称 | Agent数 | 成本 | 适用场景 |
|:----:|:-----|:-------:|:----:|:---------|
| **L1** | 🏃 独行 | 1 | 1× | 快速原型、简单脚本 |
| **L2** | 🤝 搭档 | 2 | 2× | 标准任务、基本审查 |
| **L3** | 👥 团队 | 3 | 4× | 中等复杂度、完整流程 |
| **L4** | 🔍 审计 | 4 | 6× | 重要交付、安全敏感 |
| **L5** | 🔀 双路 | 6 | 8× | 高风险、不确定最优方案 |
| **L6** | ⭐ **三路** | 9 | 12× | **最高质量** *（推荐）* |
| **L7** | 🏆 交叉 | 12 | 16× | 安全关键、不能出错 |
| **L8** | 💎 钻石 | 13 | 20× | 只接受完美 |

<div align="center">

> **默认推荐 L6 三路分流** — 3个Agent同时探索3条不同优化路径。
> Path A 优化*性能*，Path B 优化*可维护性*，Path C 优化*简洁性*。
> 对比评分，选最优方案，或融合三者精华。

## 🏆 为什么选择 Order Your Milk？

### 1. 🧋 奶茶店点单 — *市场上没有的功能*

所有AI编程工具都假设用户知道自己要什么。**这个skill引导你。** 结构化选项引导，3个问题把"我想做个网站"变成明确需求。

### 2. 🔬 三路并行调研 — *比 Deep Research 更深*

Perplexity/OpenAI 的 Deep Research 是串行的。本skill从 **3个角度同时钻探**：核心概念 + 实际应用 + 常见陷阱。配合空腔检测和自动绕行。

### 3. ⚡ 三路分流执行 — *系统性多样性注入*

没有框架原生支持"同一任务走多条路"。每条路径被赋予 **不同的优化目标**，确保方案真正不同。不是简单重复——是系统性探索。

### 4. 🌍 跨Agent通用 — *零锁定*

纯方法论，无API依赖。Hermes（完整并行）、Claude Code（串行模拟）、Cursor/Windsurf（串行模拟）、Copilot，或任意支持系统提示的Agent 都能工作。

### 5. 🎚️ 八档控制 — *精确的成本/质量旋钮*

从 1× 到 20× token 消耗。没有其他系统提供这种粒度的控制。让投入匹配任务重要性。

## 🚀 快速开始

<details>
<summary><b>🟣 Hermes Agent</b> — 完整并行，八档全可用</summary>

```bash
hermes skill install order-your-milk
```

完整并行能力——所有8个档位均可使用。

</details>

<details>
<summary><b>🟠 Claude Code</b> — 串行模拟 + --print 并行</summary>

```bash
# 将方法论复制到项目中
cp order-your-milk/SKILL.md ./CLAUDE.md
```

串行模拟三路分流。可用 `--print` 模式并行执行。

</details>

<details>
<summary><b>🔵 Cursor / Windsurf</b> — Agent模式 + 配置模板</summary>

```bash
# 从 references/platform-adapters.md 复制 .cursorrules 模板
# 或将 SKILL.md 内容复制到 .cursorrules
```

Agent模式串行执行。配置模板已包含在项目中。

</details>

<details>
<summary><b>🟢 GitHub Copilot</b> — Chat + @workspace 上下文</summary>

```bash
# 从 references/platform-adapters.md 复制配置模板
cp order-your-milk/references/platform-adapters.md .github/copilot-instructions.md
```

</details>

<details>
<summary><b>⚪ 任意 Agent</b> — 复制系统提示即可</summary>

将 `SKILL.md` 内容复制到 Agent 的系统提示中。核心方法论在任何平台都能工作。

</details>
